Detailed Description

This type decorates a pair with fallback/caching logic.

csalt_store_read() tries reading the first store first. If all the requested data are read, it returns immediately. Otherwise, it attempts to read the remaining data from later stores. If data is read from the later stores successfully, it is written back into earlier stores.

If any store returns an error code from csalt_store_read(), the whole fallback store halts immediately and the contents of *buffer is undefined.

csalt_store_write() writes only to the first store in the list. The data can be written out to all stores by calling csalt_store_flush() on the list.

csalt_store_size() implements the same logic as for csalt_store_pair.

csalt_store_split() behaves similarly to csalt_store_pair, except the result implements the same fallback logic for csalt_store_read() and csalt_store_write().

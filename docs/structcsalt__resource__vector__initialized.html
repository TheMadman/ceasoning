Detailed Description

A type representing an initialized vector, implementing the csalt_store interface.

Writes and splits past the end of this type will attempt to resize the pointed-to heap memory before writing.

Initialization and clean-up are performed by csalt_resource_vector, which is how you should use this type. Otherwise, you must perform clean-up of the internal pointer yourself, which you should avoid.

size_t csalt_resource_vector_initialized::amount_written

Stores the amount of data written, to prevent reading uninitialized values.

If you split/write after the beginning, then read from the beginning, the memory before your write will be initialized to 0.

◆ original_pointer

void* csalt_resource_vector_initialized::original_pointer

This pointer is the pointer to the initialized heap memory, as returned by malloc() or realloc(), and is the member that needs free() called on it.

Again, you should only use this type via csalt_resource_vector, which will do the initialization and clean-up for you when passed to csalt_resource_use().

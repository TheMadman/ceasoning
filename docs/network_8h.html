Provides an alternative interface for getaddrinfo.

This interface removes the need to perform resource management for the getaddrinfo resource returned, by instead using a callback.

If there is a getaddrinfo error, the callback is not called.

Parameters
nodeThe node to get the address for, usually an IP address or DNS domain name
serviceThe service to get the address for, usually a port number or one of the human-readable service names in the system's "services" file, e.g. /etc/services
hintsAn addrinfo struct, used to narrow down the types of addresses to query
callbackA function for using the resulting addrinfo linked list
paramAn additional parameter to pass to the callback
Returns
The return value of callback on success, otherwise an error code from getaddrinfo.

Detailed Description

Treats two pairs as a single pair.

csalt_resource_init() attempts to initialize the first resource. If that returns NULL, this returns NULL. Otherwise, it attempts to initialize the second resource. If that returns NULL, the first resource is freed, then this returns NULL. If both succeed, the result is wrapped in a csalt_store_pair and returned.

csalt_resource_deinit() deinitializes the second resource, then the first resource.

This macro is the recommended way of constructing a pair_list, given the two arrays' sizes are defined at compile-time. Dynamic arrays, such as those allocated with malloc() or calloc(), must still use csalt_store_pair_list_bounds().

Creates a linked-list-like array of pairs.

This function effectively constructs a linked-list from pairs. The first pair's first property points to the first store in the array. The second property points to the second pair. The second pair's first property points to the second store, and its second property points to the next pair, and so on. The last pair's second property is set to a null pointer value.

Note that this function uses an out-parameter for initializing an already-existing array, instead of returning a value.

Constructing an array for an existing local array of store is simple:

struct csalt_resource_pair list[arrsize(resource)] = { 0 };
